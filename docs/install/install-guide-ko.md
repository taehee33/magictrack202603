# MagicTrack 설치 가이드

이 문서는 일반 사용자가 MagicTrack를 설치하고 처음 실행할 때 필요한 절차를 자세히 설명합니다.

현재 배포본은 Apple Developer ID 서명 및 notarization이 없는 빌드일 수 있습니다. 이 경우 macOS 보안 경고가 나타날 수 있습니다.

## 1. 다운로드

배포 페이지에서 아래 파일 중 하나를 다운로드합니다.

- 권장: `.dmg`
- 대안: `.zip`

## 2. 설치

### DMG 설치

1. `.dmg` 파일 더블클릭
2. 열린 창에서 `MagicTrack.app`를 `Applications` 폴더로 드래그
3. 복사 완료 후 `Applications` 폴더에서 앱 확인

### ZIP 설치

1. `.zip` 파일 더블클릭
2. 압축 해제된 `MagicTrack.app` 확인
3. `Applications` 폴더로 이동

권장:

- 꼭 `Applications` 폴더로 옮긴 뒤 실행하세요
- 다운로드 폴더에서 바로 실행하면 권한/격리 관련 문제가 더 자주 발생합니다

## 3. 첫 실행

정상적으로 열리면 바로 앱 창 또는 메뉴바 아이콘이 보입니다.

하지만 인증서가 없는 빌드는 아래 경고가 나올 수 있습니다.

예:

- 확인되지 않은 개발자
- 보안상 열 수 없음
- 손상되었기 때문에 열 수 없음처럼 보이는 경고

## 4. 가장 먼저 해볼 실행 방법

1. `Applications` 폴더 열기
2. `MagicTrack.app` 우클릭
3. `열기`
4. 경고창이 나오면 다시 `열기`

이 방식은 일반 더블클릭보다 macOS 보안 우회를 더 쉽게 허용합니다.

## 5. 그래도 실행이 안 될 때

### 방법 A: 시스템 설정에서 허용

1. `시스템 설정`
2. `개인정보 보호 및 보안`
3. 아래쪽으로 스크롤
4. `MagicTrack이 차단됨` 또는 유사 문구 확인
5. `그래도 열기`

그 다음 다시 앱 실행

### 방법 B: 우클릭 > 열기 다시 시도

일부 경우:

1. 한 번 차단된 뒤
2. 다시 `Applications` 폴더에서 우클릭 > 열기

로 통과되는 경우가 있습니다.

## 6. 첫 실행 후 권한 설정

MagicTrack는 일부 기능을 위해 macOS 권한이 필요할 수 있습니다.

### 입력 모니터링

필요한 경우:

- 트랙패드 설정 변경 관련 기능

경로:

1. `시스템 설정`
2. `개인정보 보호 및 보안`
3. `입력 모니터링`
4. `MagicTrack` 허용

### 손쉬운 사용

필요한 경우:

- 앱 자동전환 기능

경로:

1. `시스템 설정`
2. `개인정보 보호 및 보안`
3. `손쉬운 사용`
4. `MagicTrack` 허용

권한 변경 후 앱을 다시 실행하는 것이 안전합니다.

## 7. 설치 후 기본 사용법

### 트랙패드 탭

- 내장 트랙패드 설정 조정
- 매직 트랙패드 설정 조정
- 각 장치 카드에서 `이 설정 적용`

### 프리셋 탭

- 현재 설정 저장
- 저장된 프리셋 적용

### 설정 탭

- 앱 자동전환 켜기/끄기
- 실행 중 Dock에 보이기
- 실행 중 메뉴바에 보이기

### 빠른 전환

- 단축키: `option + M`
- 내장 / 매직 트랙패드를 빠르게 전환

## 8. 현재 제한 사항

사용 전 아래 내용을 꼭 알아두는 것이 좋습니다.

- Apple 트랙패드의 실시간 실입력 감지는 현재 미지원
- 좌표/제스처 감지는 미지원
- 입력 기반 자동화는 미지원
- 매직 트랙패드 회전은 현재 실험 기능
- Apple 플랫폼 제약으로 완전한 실시간 독립 감도 제어에는 제한이 있음

즉 현재 버전은:

- 장치별 설정 저장
- 수동 적용
- 빠른 전환

중심으로 쓰는 것이 맞습니다.

## 9. 설치 문제 해결 체크리스트

문제가 있으면 아래 순서로 확인하세요.

1. `Applications` 폴더에서 실행했는지
2. 우클릭 > 열기로 시도했는지
3. `개인정보 보호 및 보안 > 그래도 열기`를 눌렀는지
4. 입력 모니터링 / 손쉬운 사용 권한을 허용했는지
5. 앱을 완전히 종료 후 다시 실행했는지

## 10. 자주 묻는 질문

### Q. 왜 일반 앱처럼 바로 안 열리나요?

현재 배포본에 Apple Developer ID 서명 및 notarization이 없으면 macOS 보안 경고가 더 강하게 뜹니다.

### Q. 앱이 악성 앱인가요?

아닙니다. 다만 macOS가 서명되지 않은 앱을 기본적으로 더 엄격하게 차단하는 구조입니다.

### Q. 앞으로 더 쉽게 설치되게 바뀌나요?

가능합니다. 배포자가 Apple Developer Program 가입 후 `Developer ID Application` 인증서와 notarization을 적용하면 설치 경험이 개선됩니다.
